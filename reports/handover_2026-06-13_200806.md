## Problems / Solutions

- **Mono-provider regression after Fable block.** Solved structurally: cross-provider at design (SHIP-1), verification (SHIP-2), and review (SHIP-3) — not by restoring a code-writing model.
- **`append_jsonl` is `CLAUDE_HOME`-bound** (caught by SHIP-1 challenge at design time). The AC's "reuse append_jsonl" + "support `--log-path`/`CLAUDE_BOOSTER_LOGS_DIR` override" were mutually exclusive. Lead resolved it pre-code: `kpi_rework.py` owns its log path; reuses only `iso_now`/`project_root_from`.
- **`.go_active` drift** — deployed `go.md` had dropped the marker writes go_gate depends on; gate was dead. Restored via template→deployed sync.
- **Codex CLI portability** — SHIP stages were Claude-centric. Added the provider-symmetric "Cross-provider stages" adapter to the bridge skill; `kpi_rework.py`/`phase.py`/`model_balancer.py` are plain python (auto-deployed) so identical on both CLIs.
- **Recursion risk** in "post-`/go` runs `/debt auto`, which runs `/go`" — solved with the `.debt_auto_active` guard marker; nested `/go` skips its post-step.
- **`git commit -m "...`backticks`...$var..."` corrupted a commit message** (shell ran the backticks, expanded `$RUNTAG`). Fixed by amending via `git commit -F <file>`; lesson saved to memory.
