## Problems / Solutions

| Problem | Solution |
|---|---|
| “Fable as Lead” would burn tokens and blur orchestration with judgment. | `/go fable` defines Fable as Quality Chair only; Booster Lead still controls orchestration, evidence, retries, git, and verdict. |
| Context could be lost between early Fable Challenge and late Fable Diff-review. | `fable_control.watchlist` and `rework_log` persist context inside the augmented PFD; no hidden chat memory is required. |
| Fable could create open-ended review loops. | Hard cap: two Fable calls per run; post-rework review uses the normal reviewer against the persisted watchlist. |
| Fable review could override the exit-code verdict. | Spec says Fable can route typed rework but cannot mark final PASS; PASS remains the Verifier test exit code. |
| Codex bridge would not advertise the new mode. | Updated skill, prompt, booster-command runner, and `AGENTS.md`; bridge integration still passes. |
