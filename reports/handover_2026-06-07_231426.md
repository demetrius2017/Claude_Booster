## Problems / Solutions

### Problem: Codex cannot be promised custom bare slash commands

Codex has built-in slash commands, but no stable custom top-level slash-command
registry equivalent to Claude Code command files.

Solution: use Codex skills as the primary UX:

```bash
$audit <topic>
$consilium <topic>
$handover
$architecture [--update]
```

Legacy prompt aliases were also installed:

```bash
/prompts:consilium <topic>
```

### Problem: initial SKILL.md frontmatter had invalid YAML

Unquoted descriptions with colons caused Codex to skip several skills.

Solution: quote all relevant frontmatter values and add installer validation.

### Problem: first installer was a blind copy into HOME

The first version could overwrite user-owned skills, leave stale files, and
report success without checking destination state.

Solution: rewrite installer with:

- stdlib frontmatter validation
- symlink rejection
- command-spec preflight
- collision preflight
- backups
- atomic writes
- manifest ownership
- stale cleanup
- destination hash verification
- local installed command specs under `booster-command/references/commands`

### Problem: audit protocols referenced Claude Agent/PAL semantics

Codex does not always expose the same tools.

Solution: `booster-command` now requires honest labelling:

- PAL if PAL MCP exists.
- Codex second-opinion subagent if subagents exist.
- Local fallback only if neither exists; do not label it as full Booster
  multi-agent parity.
