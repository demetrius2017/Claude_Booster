## Problems / Solutions

- **P: I claimed `docs/dep_manifest.json` was absent in the `/start` Context Receipt — without running `ls`.** S: It exists (35 KB). Caught by the brief-verifier, re-verified by me. This is the exact failure `code-over-docs.md` exists to prevent, and I committed it first in the session. An unverified line in a Context Receipt is **worse than a missing one**, because downstream agents treat it as established fact.

- **P: Consilium #1's headline conclusion rested on a units error.** S: "35 of 58 runs" put defect *instances* next to defect *runs* in adjacent sentences. Real figure: 20/58 runs (34%). Corrected in consilium #2 with the full table. New rule: **when a brief cites a number, it must state the unit.**

- **P: Three separate instruments lie the same way.** S: (1) `access_count = 0` on every memory row means *uninstrumented*, not unread — `build_context()` never calls `recall()`, `_fetch_start_context()` sits on a read-only connection and physically cannot write. (2) `kpi_rework` measures retries *within one `/go` run*, never the cross-session production loop. (3) `model_balancer.json` holds a frozen `claude_max_weekly_used_pct: 0.86` (no `weekly_pct_source`, `weekly_tokens_cap` unset → live path dead), while the live `.rate_limits_cache.json` says `seven_day_remaining: 86` — **two "86"s meaning opposite things, coinciding by accident.** Had I trusted the balancer I would have denied Fable citing an exhaustion that does not exist. Fixed #1 this session; #2 is Phase 2; #3 is logged as debt.

- **P: My prototype plan was a fiction of proof.** S: It captured a "golden" render from the **live** DB, then re-captured after the edit — but this session's own hooks mutate that DB between captures. The byte-diff would have shown data drift, not code change. The Fable Challenge caught it pre-code. The Prototype Gate rebuilt it on a **pinned frozen copy** (monkeypatching `rolling_memory.DB_PATH`, the only mechanism — no env override; `rolling_memory.py:50`, read at call time by `:354` and `:372`). Only then was the invariant provable.

- **P: A top-level `import memory_telemetry` in `rolling_memory.py` would have been catastrophic.** S: `rolling_memory` is `critical: true` and **every hook imports it**. Any defect in the new module — syntax error, partial install, missing file — would have taken down cross-session memory on every project, to buy one log line. Fable Challenge caught it (FBL-001). Fixed: lazy guarded import; test `missing_telemetry_module_is_safe` stands watch.

- **P: `_gate_common.append_jsonl()` cannot provide fail-open.** S: It catches only `OSError` (`_gate_common.py:103`); any other exception propagates to the caller. `memory_telemetry.py` therefore ships its own writer wrapped in a swallowed `except Exception` (never `BaseException` — `KeyboardInterrupt`/`SystemExit` must still propagate). Proven by three **real induced failures**, not by reading the try/except.

- **P: `build_start_context` has three return points, not one.** S: Found by the Prototype Gate — nobody else saw it. Instrument only the happy path and the telemetry silently loses whole branches, undetectably, because telemetry *is* the instrument. All four return points (3 + `build_context`'s one) now emit.

- **P: `report` would have silently created an empty DB.** S: `sqlite3.connect()` on a missing path creates a 0-byte file — a documented institutional defect. Now `file:{path}?mode=ro` + an `exists()` guard. Test `report_does_not_create_db` PASS.

- **P: 4 `.sh` tests are red.** S: **Not a regression.** Verified RED at `HEAD` on a clean `git worktree` before the change. Logged as adjacent debt rather than silently absorbed.

- **P: GLM-5.2 unavailable in both consilia.** S: Three attempts each, all `529`. Slot recorded as `DEGRADED` and **not substituted** with another model — substituting would have been a lie about provider diversity. This is exactly the degradation v1.17's health-aware routing was built for.
