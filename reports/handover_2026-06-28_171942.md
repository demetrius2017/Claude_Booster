## Problems / Solutions

Problem: Z.ai route existed, but the user's own shell could not call it because `ZAI_API_KEY` was absent.

Solution: `zai_cli.py` now reads `ZAI_API_KEY` first and falls back to `~/.claude/secrets/zai_api_key`. The file is local, chmod `0600`, and `~/.zshenv` exports it for new shells without storing the secret value in config.

Problem: Reviewer found that `templates/scripts/zai_cli.py` was missing from the tracked diff.

Solution: The runner is now added to the working tree and marked as a new tracked candidate.

Problem: Reviewer found that same-day `model_balancer.json` files did not persist newly merged default routes.

Solution: `model_balancer.decide()` now detects `_with_default_routes()` changes and writes the merged JSON even when the file is already fresh for today.

Problem: Grok CLI needed to be both reviewer and coder, but direct write access to the main checkout would raise blast radius.

Solution: Split Grok into two paths:
- `grok_cli.py` for read-only smoke/review.
- `grok_sandbox_worker.sh` for coding in a detached git worktree, returning only a diff.

Problem: `/audit` smoke originally only covered PAL/Z.ai/DEGRADED.

Solution: Extended it to cover `grok_cli.py review --budget-turns 3` and DEGRADED wording that includes Grok unauthenticated.
